# MJC in one — 구현 진행 요약

코드베이스(`app/lib`, `main.dart`, `pubspec.yaml`, `test/`) 기준으로 정리했습니다. 배포·운영 환경(Firebase 콘솔, 스토어 심사 등)은 포함하지 않습니다.

---

## 완성됨

앱의 핵심 사용자 흐름이 동작하는 수준으로 구현된 항목입니다.

- **앱 셸·내비게이션**  
  - 하단 탭 + FAB 확장 메뉴, `AnimatedSwitcher` 전환  
  - 스캐폴드 드로어 + 홈 전용 슬라이드 메뉴(엣지 제스처·스냅)  
  - 시스템 뒤로가기: 드로어·메뉴·탭 히스토리·종료 순 처리

- **홈 대시보드**  
  - Firestore 경로를 묶는 `NoticeManager`로 **최근 2주** 통합 피드(공지·학사·장학·역량·CTL 등)  
  - 당겨서 새로고침, 공지 카드·상세 웹뷰 이동, MPU 바로가기 등

- **메인 홈페이지 / CTL / MPU 화면**  
  - 탭별 공지·프로그램 목록, `NestedScrollView` + 새로고침  
  - 항목 탭 시 `CommonWebViewScreen`으로 원문 URL 표시  
  - 목록 입장 애니메이션(`flutter_animate`) 등 UX 마감

- **도서관**  
  - `lib.mjc.ac.kr` 인앱 웹뷰, 뒤로/앞으로, 새로고침, 공유·외부 브라우저

- **푸시 알림 (클라이언트)**  
  - Firebase 초기화, 권한 요청, `all_notices` 토픽 구독  
  - 포그라운드·백그라운드 수신 후 로컬 알림 표시(Android 채널)  
  - 전체 알림 / 키워드 필터, 출처(MJC·CTL·MPU) 스위치 → `SharedPreferences` 저장

- **알림 내역**  
  - 수신 알림 로컬 저장(최대 50건), 목록·개별 삭제·전체 삭제  
  - 드로어 상단 미리보기(최신 3건) 연동

- **설정·정보**  
  - 알림·키워드·출처 설정, 개발자 메일 문의(`mailto`), 오픈소스 라이선스 전용 화면(`LicenseRegistry` 기반)

- **테마·UI 공통**  
  - `AppColors` / `buildMjcTheme()` 기반 색·타이포 정리

- **저장소 외 작업물**  
  - `test/` 크롤러·샘플 HTML 등 데이터 수집 스크립트(앱과 별도 파이프라인)

## 보완 필요

동작은 하나, 품질·플랫폼·유지보수 측에서 손보면 좋은 항목입니다.

- **웹 빌드**  
  - `main.dart`에서 Firebase 미구성 시 try-catch로 넘어가므로, 웹에서는 푸시·Firestore 의존 기능이 제한되거나 비어 보일 수 있음.

- **로컬 알림 초기화**  
  - `FlutterLocalNotificationsPlugin` 초기화가 Android 중심으로 보이며, iOS 등 다른 플랫폼 알림 설정이 코드상으로는 얇음.

- **데이터 레이어**  
  - `FirestoreService`(스트림 구독)는 정의만 있고 앱 화면에서 사용되지 않음. 실시간 갱신·오프라인 정책을 쓰려면 통합 또는 제거 결정이 필요.

- **날짜·통합 로직**  
  - `NoticeManager`의 날짜 파싱은 정규식 기반이라 비표기 포맷은 대시보드 2주 필터에서 누락될 수 있음.

- **하드코딩**  
  - 앱 이름·버전 문자열이 드로어·설정·`showAboutDialog` 등에 분산(예: `1.0.0`). 단일 소스(`package_info_plus` 등)로 맞추면 유지보수에 유리.

- **도움말·온보딩**  
  - 도움말은 짧은 안내 다이얼로그 수준; 첫 실행 가이드·FAQ는 없음.

- **테스트**  
  - `widget_test.dart` 수준; 비즈니스 로직·화면 통합 테스트는 거의 없음.

## 미완성(예정)

루트 `README.md`의 비전(통학·캠퍼스 이동 연계 등) 대비 앱 코드에 아직 없거나, 별도 백엔드·제품 작업이 필요한 항목입니다.

- **통학·교통·캠퍼스 이동** 등 README에 언급된 확장 기능 — 현재 Flutter 앱 코드에 해당 모듈 없음.

- **스토어 배포·CI·버전 자동화** — 저장소 기준으로는 일반 템플릿 수준(`app/README.md`).

- **푸시 탭 시 상세 화면/딥링크** — 수신·로컬 표시·내역 저장은 있으나, 알림 탭으로 특정 공지 URL을 여는 통합 플로우는 코드에서 명확히 보이지 않음.

- **앱 전역 검색·즐겨찾기·계정 연동** 등 — 미구현.

- **크롤러→Firestore 파이프라인 운영** — 스크립트는 있으나, 스케줄·모니터링·장애 대응은 제품화 범위 밖으로 가정.
